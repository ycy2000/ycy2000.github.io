1. byte변수형 결과, round, ★int, fix

  • byte(0에서 127까지 128개 정수) 변수의 결과
    • Dim b As Byte
    b = 1 + (3 / 8) '결과 : 1, 0.5이하일때 정수부분만
    b = 1 + (4 / 8) '결과 : 2, 0.5포함 이상일때 반올림됨
  • 양수 round(숫자, 소수점자리수) : 자리수는 정수자리 끝을 0 으로보고 결과자리수이며 한자리 아래것을 반올림한다.
    • round(12.135, 0) => 결과정수끝0자리 : 오른쪽소수점1자리가 1이므로 반올림 X ==> 12
    round(12.135, 1) => 결과소수점1자리 : 오른쪽소수점2자리가 3이므로 반올림 X ==> 12.1
    round(12.135, 2) => 결과소수점2자리 : 오른쪽소수점3자리가 5이므로 반올림 O ==> 12.14
  • 음수 round는 부호만 다를뿐 결과는 같다고 보면 되겠다.
    • 음수 round(숫자, 소수점자리수) : 자리수는 정수자리 끝을 0 으로보고 결과자리수이며 한자리 아래것을 반올림한다.
    round(-12.135, 0) => 결과정수끝0자리 : 오른쪽소수점1자리가 1이므로 반올림 X ==> -12
    round(-12.135, 1) => 결과소수점1자리 : 오른쪽소수점2자리가 3이므로 반올림 X ==> -12.1
    round(-12.135, 2) => 결과소수점2자리 : 오른쪽소수점3자리가 5이므로 반올림 O ==> -12.14
    ※ 양수int와 음수int가 결과가 다르다 주의 필요
    양수 : int(12.3) => 정수부분만 결과로 12
    양수 : int(12.9) => 정수부분만 결과로 12
    음수 : int(-12.1123) => -13 : 음수일경우 소수점값이 0이 아니면 정수부분이 +1 된다.
    음수 : int(-12.0009) => -13 : 음수일경우 소수점값이 0이 아니면 정수부분이 +1 된다.
    ※ fix : 양수든 음수든 소수점값 상관없이 정수부분만 취할때
    양수 : fix(12.3) => 정수부분만 결과로 12
    양수 : fix(12.9) => 정수부분만 결과로 12
    음수 : fix(-12.3) => 정수부분만 결과로 -12
    음수 : fix(-12.9) => 정수부분만 결과로 -12

    2. 컨트롤(textbox많고 폼켤때 시트참조 정보가 들어가야 할 때 코딩 참고)

  • 폼내 모든 컨트롤을 돌면서 컨트롤이름 참고하여 정보 가지고오기
    • Private Sub UserForm_Initialize()
    Sheets("sheet1").Select
    Dim 화주기준셀 As Range
    Dim 품명기준셀 As Range
    Dim 창고기준셀 As Range
    Set 화주기준셀 = Range("a4")
    Set 품명기준셀 = Range("b4")
    Set 창고기준셀 = Range("c4")
    Dim 컨트롤 As Control
    For Each 컨트롤 In Me.Controls
    If Left(컨트롤.Name, 2) = "화주" Then
        컨트롤.Value = 화주기준셀.Offset(Right(컨트롤.Name, 3) * 1, 0)
    End If
    If Left(컨트롤.Name, 2) = "품명" Then
        컨트롤.Value = 품명기준셀.Offset(Right(컨트롤.Name, 3) * 1, 0)
    End If
    If Left(컨트롤.Name, 2) = "규격" Then
        컨트롤.Value = 창고기준셀.Offset(Right(컨트롤.Name, 3) * 1, 0)
    End If
    Next
    End Sub

    3. MultiPage

  • 연속탭(TabStrip)과 멀티페이지 차이 (멀티페이지만 쓴다)
    • 멀티 페이지는 컨트롤 등을 담는 일종의 컨테이너 같은 역할(Frame과 같이.)을 하므로
    페이지를 바꾸는 데에 있어서 걱정을 할 필요가 없습니다.(독립적)
    그냥 상식대로... 엑셀 Sheet마다 다른 내용을 담듯이 UserForm 멀티 페이지에도
    각 페이지마다 담고 싶은 내용을 담으면 알아서 작동을 해줍니다.

  • 다중페이지 조작하기
    • ▸ 페이지 선택
        multipage1.value : 0 , 순번 : 0이 첫번째이고 첫번째가 선택된 상태라는 의미
        multipage1.value = 1 , 두번째 page를 선택하라는 의미 (초기화때 등)
        multipage1.value = 0 : 첫번째 페이지는 선택하라는 의미
    ▸ page2에 접근 : 이름으로 접근
        me.multipage1.page2.name : 이름바뀌면 바뀐이름.. (화면에 보이는 것은 caption이니 혼동하지 말것)
    ▸ page2에 접근 : pages(인덱스번호 : 0부터시작) 으로 접근
        me.multipage1.pages(1).name : 두번째 페이지의 이름
    ▸ 두번째 페이지 안의 컨트롤만 확인할때
        For Each 컨트롤 In Me.MultiPage1.Pages(1).Controls
    ▸ page2 안의 textbox1의 표현
        1.(0) : textbox1 : 컨트롤이름 그대로
        2.(X) : me.multipage1.textbox1
           (단계적 접근시 모든 경로 표시해야함)
        3.(0) : me.multipage1.page2.textbox1 (이름으로)
        4.(0) : me.multipage1.pages(1).textbox1 (index번호로)
